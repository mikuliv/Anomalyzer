# Anomalyzer

> Архивный проект. Активная разработка завершена.

## О проекте

Anomalyzer — историческая система обнаружения аномалий сетевого трафика, созданная как выпускная квалификационная работа и опубликованная под GNU GPLv3.

Репозиторий переведён в архивное состояние. Исправления, обновления зависимостей и поддержка не гарантируются.

## Возможности исторической версии

Историческая реализация классифицирует сетевые потоки по 78 числовым признакам, извлечённым из CICIDS2017. В состав реализации входят MLP-модель в формате ONNX, FastAPI API, Logstash и визуализация через Kibana.

## Запуск

Исторический запуск требовал Docker и Docker Compose:

```powershell
cd runtime-system
docker-compose up --build
```

- `ml-api`: `http://localhost:8000/predict`
- Logstash читает CSV из `logstash/data/*.csv`
- Kibana: `http://localhost:5601`

Эта команда не проверяется и не поддерживается как современный production workflow.

## Используемые технологии

- PyTorch и ONNX;
- FastAPI;
- scikit-learn, включая LabelEncoder, SMOTE и масштабирование;
- Elasticsearch, Logstash и Kibana;
- Docker Compose.

## Данные и модели

Историческая работа использует сторонний датасет [CICIDS2017](https://www.unb.ca/cic/datasets/ids-2017.html). Сам датасет может не входить в Git. Локальные historical models, scaler и список классов относятся к архивной реализации; их наличие не означает передачу прав на исходный dataset. Использование данных и моделей должно учитывать условия их источников.

Исторически зафиксированные в документации метрики: accuracy `0.987`, F1-score `0.986`, ROC-AUC `0.993`, precision `0.988`, recall `0.984`.
