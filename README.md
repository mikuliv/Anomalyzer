# Anomalyzer / Филин

Репозиторий содержит историческую версию проекта Anomalyzer и новую ветку развития платформы Филин.

## Структура

- `Anomalyzer-main/` - базовая версия проекта Anomalyzer, выполненная как выпускная квалификационная работа СПО по специальности 10.02.05 «Обеспечение информационной безопасности автоматизированных систем».
- `filin/` - новая версия платформы для развития проекта в рамках ВУЗа.

## Цель развития

Цель развития проекта - переход от простого ML-классификатора сетевых аномалий к платформе мониторинга, анализа и поддержки расследования инцидентов информационной безопасности.

Новая платформа должна объединить сбор сетевых событий, извлечение признаков, ML-детекцию, формирование карточек инцидентов, сопоставление с MITRE ATT&CK, подготовку Sigma-кандидатов и проверку правил на лабораторном стенде.

## Логика перехода

Anomalyzer сохраняется в репозитории как историческая база: он показывает исходную постановку задачи, начальную ML-модель и демонстрационный контур дипломного проекта СПО. Эта часть не используется как место для текущей разработки.

Филин развивается отдельно как самостоятельная университетская платформа. Новая часть проекта должна расширить исходную идею в сторону инженерной архитектуры, воспроизводимого ML-пайплайна, обработки инцидентов, интеграции с правилами обнаружения и подготовки к подключению SIEM-инструментов.

Основное направление развития:

- нормализация сетевых событий из разных источников;
- выделение признаков для моделей обнаружения;
- прототипирование API детекции на FastAPI;
- создание карточек инцидентов для аналитика;
- сопоставление классов атак с MITRE ATT&CK;
- генерация черновиков Sigma-правил;
- проверка правил и сценариев на лабораторном стенде.
